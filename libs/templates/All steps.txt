STEP0 - How to Navigate the Project (Backend)
1) Start point for API routes
apps\master-management\src\main.ts
Global prefix is /api

2) Check module wiring
apps\master-management\src\app\app.module.ts
Look for LeadModule, AuthModule, UserManagementModule

3) Lead + Service API entry points
apps\master-management\src\app\lead\lead.controller.ts
POST /api/leads
POST /api/leads/services
GET /api/leads
GET /api/leads/services
GET /api/leads/:id
PATCH /api/leads/:id

4) Lead + Service business logic
apps\master-management\src\app\lead\lead.service.ts
createLead, updateLead, createService, getServices

5) Lead DTO structure
libs\dtos\master_management\lead.dto.ts
CreateLeadDto, UpdateLeadDto, CreateServiceDto

6) Core entities and relationships
libs\database\src\entities\lead.entity.ts
libs\database\src\entities\lead-service.entity.ts
libs\database\src\entities\customer.entity.ts
libs\database\src\entities\customerAddress.entity.ts
libs\database\src\entities\customerContact.entity.ts

7) Sales workflow APIs
apps\sales\src\app\lead\lead.controller.ts
POST /api/leads/create
PATCH /api/leads/service/:id

apps\sales\src\app\proposal\proposal.controller.ts
POST /api/proposals
GET /api/proposals/:id
GET /api/proposals/lead/:leadId

apps\sales\src\app\closure\closure.controller.ts
POST /api/closure

8) Sales workflow logic
apps\sales\src\app\lead\lead.service.ts
apps\sales\src\app\proposal\proposal.service.ts
apps\sales\src\app\closure\closure.service.ts

9) Enquiry ID rules
Master Management lead uses ENQ-{timestamp} in lead.service.ts
Sales uses IS/YY/MM/DD/sequence in lead.service.ts

STEP0.2 - How to add a new template file
1) Create the text file under libs\templates
2) Use this format:
TITLE
Step 1
Step 2
Step 3
3) Keep each section short and action oriented
4) Link every step to the exact API and file path

Step 1 → Lead (Client Registration)
Step 2 → Services (Requirement & Ownership)
Step 3 → Proposal (Commercials)
Step 4 → Closure (Acceptance, Billing & Handover)
